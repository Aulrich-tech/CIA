Lesson One

Firewalls

What Is a Firewall?

As networks began to grow, interconnect with one another, and eventually connect to the internet, it became important to control the flow of network traffic. Firewalls became a means of control that had to evolve and change alongside networks. They are classified into generations: first-generation (stateless), second-generation (stateful), third-generation (application layer), and next-generation firewall (NGFW).

Four Generations of Firewalls
Generation Also Known As How It Works Key Limitation
1st Gen Stateless / Packet Filter Firewall Examines routing & transport layer info: source/destination IP, protocols, port numbers. Rules are ordered top-to-bottom; last rule is implicit deny or explicit action. Requires extra rules for return traffic; cannot manage protocols that open random ports (e.g., FTP). Uses a "one-size-fits-all" approach; bad actors can inject rogue packets via acceptable ports.
2nd Gen Stateful Firewall Tracks the 5-tuple check and connection state in a session table. Monitors traffic going back and forth between endpoints. Blocks connections that behave improperly or whose return traffic doesn't match incoming traffic. Cannot block rogue packets if they use an acceptable protocol (e.g., HTTP). Cannot distinguish between different uses of the same protocol running on the same port.
3rd Gen Application Layer / UTM Firewall Performs Application Layer Filtering (ALF) — understands protocols at the application layer such as HTTP, FTP, DNS. Can distinguish between blog traffic, e-commerce, VoIP, email, etc. UTM adds antivirus, antispam, IPS, VPN. IT security teams managing application learning cannot keep pace with rapid mutation of exploits; no native defense against zero-day attacks.
4th Gen / NGFW Next-Generation Firewall Rule-based packet inspection + DPI (deep packet inspection) + sandboxing + AI for proactive security. Supports network segmentation to eliminate single points of entry. Delivers high-performance inspection with minimal network degradation. More complex to configure; requires skilled administrators. Broader feature set means larger attack surface if misconfigured.
Stateless Firewall — Deep Dive

How Stateless Firewalls Work

A stateless firewall allows a packet to pass if the network addresses, protocol, and port number match its firewall policy. If it does not match, the packet is either silently dropped or blocked.

Policies are ordered in a list; potential matches are evaluated top-to-bottom. The last firewall policy can be implicit (deny by default) or explicit (perform configured action).

Key weakness: Requires additional configuration for return traffic. Stateless firewalls open random ports and use multiple connections (like FTP control + data). Bad actors can bypass rules using acceptable protocols or exploit bugs in networking software.

How Stateful Firewalls Work

A stateful firewall observes network connections over time by tracking the 5-tuple check (source IP, destination IP, source port, destination port, protocol) and the connection state in its session table.

It watches as new connections are made and continuously examines traffic going back and forth between endpoints. If a connection behaves improperly or return traffic doesn't match the incoming traffic, the firewall blocks that connection.

Any packet that doesn't belong to a known conversation or doesn't match an allowed policy is dropped.

Application Layer & Next-Gen Firewalls

Why HTTP Was a Problem

The explosion of the World Wide Web made HTTP one of the most frequently used network protocols. HTTP is used in many ways: static text, e-commerce, file hosting, web apps — all on the same port number. Stateful firewalls couldn't distinguish between them.

Network admins needed to distinguish approved from malicious applications. Firewalls had to look deeper into data payloads — this led to application layer filtering.

UTM Firewall Features

Third-generation firewalls that implement application layer filtering understand protocols like HTTP, FTP, and DNS. HTTP-aware firewalls can distinguish between browser traffic, a blog, file sharing, e-commerce, social media, VoIP, and email.

UTM firewalls also combine additional protections:

Antivirus Antispam IPS VPN

Next-Generation Firewall (NGFW) — The Airport Analogy

A next-generation firewall operates like airport security, with multiple security checkpoints:

Boarding Pass Check
Like a security agent checking your pass, the NGFW looks at packets and makes rule-based decisions whether to allow or drop traffic.
Bag Screening (DPI)
Security checks bags for banned items — similarly, the NGFW performs deep packet inspection (DPI) to look inside the packet payload for malicious content.
Enhanced Screening (Sandbox)
If suspicious items are found, the bag is set aside for further review — similarly, the NGFW sends malicious content to a sandbox for further analysis.
NGFW evolution: As challenges grow, NGFWs control applications by classification or user identity. They use segmentation to eliminate single points of entry, leverage AI to enforce security policies, and deliver high-performance inspection for hybrid data center infrastructure.

Lesson Two

Network Access Control (NAC)

What Is NAC?

Network Access Control (NAC) is an appliance or virtual machine that controls access to a network. It has complete visibility into network profiles and categorizes devices automatically. It evaluates and classifies security-policy compliance by user, device, location, operating system, and other criteria that may detect unusual activity. Many NAC solutions have centralized architecture that improves control of devices across large and multi-site networks.

NAC Origins — IEEE 802.1X Authentication

IEEE 802.1X Authentication Model

NAC began as a network authentication and authorization method for devices joining the network, following the IEEE 802.1X standards. The authentication method involves three parties:

Client Device
Provides credentials (username/password, digital certificate, or other means) to the authenticator.
Authenticator
A network switch or wireless access point that demarks the protected from the unprotected network. Forwards credentials to the server.
Authentication Server
Verifies credentials. Depending on the outcome, the authenticator either blocks the device or grants it network access.
Captive Portal: Another method to control access to a publicly available network. Example: connecting to WiFi at a coffee shop where you must agree to legal terms before gaining access.
BYOD and IoT Challenges

BYOD Challenge

Guest access, Bring Your Own Devices (BYODs), and IoT devices have introduced complex problems for network security.

The first problem: devices trying to connect are personally owned, not company assets. The company's MIS department does not control what runs on these devices — for example, antivirus software or unsafe applications.

IoT Challenge

IoT devices are hardware with sensors that transmit data from one place to another over the internet, dramatically expanding the attack surface.

Organizations buy IoT-enabled devices from other vendors, and these devices connect back to vendor networks to provide information about product use and maintenance (e.g., a printer that auto-orders toner).

Security risk: IoT devices often lack CPU cycles or memory for authentication and security software. They identify themselves using a shared secret or unique serial number — if the secret is exposed, there is likely no way to reset it.
How NAC Works — Device Profiling & Segmentation

NAC Device Profiling

When NAC is introduced into a network, the first thing it does is create profiles of all connected devices. NAC then permits access to network resources based on the device profile, which is defined by function.

If a device does not have a defined profile or a profile that matches a particular function, it is denied entry — like granting individuals access to sensitive information based on their need to know.

Example: NAC can permit an IP camera connection to a network video recorder (NVR) server but would prevent it from connecting to a finance server. NAC directs the IP camera to its appropriate VLAN. If the camera is compromised, malware can only infect the NVR server, not the finance server.

NAC Segmentation Benefits

By segmenting networks rather than using a flat network, the firewall helps eliminate a single point of entry — which used to make it easy for cybercriminals to enter and spread threats across the network.

This also applies to contractors, partners, and guests who need to enter an organization's network — they are segmented by profile and can gain secure access while being locked out of restricted areas.

Guest Access & Temporary Users

With WiFi hotspots in public locations and company offices, there is a need for temporary guest access. NAC enables an organization to manage and authenticate temporary users and devices through a self-service portal.

User and device policies are adjusted as people, endpoints, and businesses change to ensure the network remains secure.

NAC Integration & Benefits

NAC Integration

NAC can be integrated into the security framework so that when a breach is detected, it automatically responds to notify the Security Operations Center (SOC) and coordinates with other security devices to neutralize the threat.

NAC integrates with other security products through the open/RESTful API or secure shell (SSH).

Five Benefits of NAC

Improved Security
Oversight of all devices; authenticates users and devices the moment they enter the network.
Cost Savings
Automated tracking and protection of devices at scale means fewer IT resources are needed.
Automation
Automates authentication and authorization — organizations cannot manually verify endpoints at scale.
Enhanced IT Experience
Seamless, frictionless access for users connecting to the network.
Ease of Control
Acts as a 24/7 inventory of authorized endpoints; assists with lifecycle management when devices must be phased out or replaced.

Lesson Three

Sandbox

What Is a Sandbox?

A sandbox, in the context of computer security, is a system that confines the actions of an application (such as opening a Word document or a browser) within a safe virtual environment. In a sandbox, an application is isolated so that any malicious intent can be uncovered safely, without causing harm to the network. If something unexpected or dangerous happens, it affects only the sandbox — not the other computers and devices on the network.

Sandbox technology is typically managed by an organization's information security team, but is used by network, applications, and desktop operations teams to bolster security in their respective domains.

Why Sandboxes Were Created — Zero-Day Attacks

The Zero-Day Problem

Threat actors exploit vulnerabilities in legitimate applications in order to compromise a device and from there move through the network to infect other devices. Firewalls and antivirus software can stop known threats, but they are helpless against zero-day attacks.

A zero-day attack is when a threat actor exploits an unknown vulnerability — one that the vendor has had zero days to patch. Sandboxing was created as a defense against zero-day attacks.

How Sandboxing Works

A sandbox offers an isolated virtual space that mimics computer systems and applications. It allows potential threats to run safely within these virtual systems and analyzes them.

  • If a sandbox deems a file safe, it does not take any action.
  • If it detects malicious behavior, it quarantines the file and prevents it from accessing other computers and devices on the network.
Three Generations of Sandboxing
1st Generation — Standalone Sandboxes
Early sandboxes encountered challenges sharing threat intelligence with other security devices due to security architecture dependence on standalone solutions. Standalone solutions could not easily integrate with products from other vendors, forcing the SOC to use a separate management console for each one. The process of consolidating and analyzing threat intelligence data was both challenging and time-consuming.
2nd Generation — Integrated Sandboxes
Second-generation sandboxes were better at integrating with other security devices, allowing for improved intelligence sharing. Analysts could correlate threat intelligence centrally and respond to threats from a single pane of glass. An integrated network security environment could share information with a threat intelligence service in the cloud, which could then push that intelligence to other networks.
3rd Generation — AI/ML-Driven Sandboxes
Threat actors leveraged automation and AI to accelerate development of new exploits. AI-driven attacks necessitated a third-generation sandbox based on a threat analysis standard. Third-gen sandbox devices not only help detect breaches in a timely manner — they also prevent them from happening. These solutions utilize the universal MITRE ATT&CK security language for categorizing threat models and methodologies.

Lesson Four

Web Application Firewall (WAF)

What Is a WAF?

A Web Application Firewall (WAF) is an appliance or software that monitors HTTP/HTTPS traffic and can block malicious traffic to and from a web application. WAFs target content from specific web applications at the application level, while edge firewalls form secure gateways between the LAN and outside servers at the network level.

By inspecting HTTP traffic, a WAF can stop attacks that target web application security flaws, such as SQL injection, cross-site scripting (XSS), file inclusion, and security misconfigurations.

History of WAF — From Application Firewalls to ML
1990s — Application Firewall Origins
The ancestor of the WAF was the application firewall developed in the 1990s. Although largely a network-based firewall, it could target some applications or protocols (FTP, RSH). The debut of the World Wide Web in 1991 was the "big bang" — expanding internet access also permitted bad actors to exploit it. As organizations became victims of espionage and theft, developing a defense against HTTP-based cyberattacks became a top priority.
1st Generation WAF — Blocklists & Signatures
Because all web applications used HTTP and either port 80 or port 443, WAF could not rely on traditional edge firewall methods. The first generation used blocklists and signature-based HTTP attributes to alert the firewall of an attack — making SQL injection attacks like the described example (using 2+2=4 as a valid SQL statement) no longer successful.
2nd Generation WAF — Behavior Baselines
Growing complexity of web apps made signature-based WAFs obsolete. False positives grew beyond IT teams' capacity. The next generation included behavior baselines and session monitoring with heuristics, enabling detection of variants of known signatures. However, rapid mutation of exploits and lack of zero-day defense remained problems.
3rd Generation WAF — Machine Learning
The next logical evolution was machine learning (ML). Unencumbered by human supervision, behavior analysis could be done at machine speed and could adapt to ever-changing threat attributes. Additional security features added: DDoS defense, IP reputation, antivirus, data loss prevention (DLP). WAFs could monitor HTTP traffic and stop any action violating acceptable behavior, identify users, correlate actions with permissions, and share information with other security devices.
SQL Injection — Worked Example

How SQL Injection Works

Imagine an online business. Customers log in using a typical login page (username + password), which is verified against a back-end database.

Normal flow: User John Smith types his user ID (jsmith) and password. If the password is true, he is allowed access; if false, he is not.

Attack flow: A bad actor pretending to be John Smith doesn't know the password. Instead of guessing, when asked for a password, the bad actor types abc123 or 2+2=4.

  • abc123 is identified as false — access denied.
  • 2+2=4 is a valid SQL statement — identified as true — bad actor gains access.
Defense: First-generation WAFs used blocklists and signature-based HTTP attributes to alert the firewall of such attacks, making SQL injection attacks no longer successful.

Modern WAF Capabilities

DDoS Defense IP Reputation Antivirus DLP Session Monitoring Behavior Baseline Zero-day via Sandbox ML-Powered Detection

WAFs can share information and collaborate with other security devices — firewalls, sandboxes — as part of a collective defense. Sandboxing capabilities mean suspicious material can be tested safely in isolation; zero-day signatures can be shared with other devices and uploaded to a threat intelligence center.

WAF vs. Edge Firewall

WAF
Operates at the application level. Targets content from specific web applications. Inspects HTTP/HTTPS traffic for malicious patterns, SQL injection, XSS, etc.
Edge Firewall
Operates at the network level. Forms a secure gateway between LAN and external servers. Makes decisions based on IP, port, protocol.

Secure Email Gateway (SEG)

What Is a Secure Email Gateway?

A Secure Email Gateway (SEG) is a technology solution designed to protect organizations from email-based threats and ensure the security and privacy of email communications. It serves as a barrier between an organization's email infrastructure and the external email network (the internet).

Email offers fast, inexpensive, anonymous communication — but also equips bad actors with means to spread misinformation, promote fraud, steal personal information, and distribute malware. Unsolicited email to many recipients is called spam. Spam that entices recipients to click a link or download a file is called phishing (named by America Online in 1996).

SEG Core Features

Content Filtering
Controls and manages types of content that can be accessed or shared. Uses keyword matching, regular expressions, DPI, and context-aware analysis.
DLP
Data Loss Prevention — prevents unauthorized or accidental leakage of sensitive/confidential information.
Spam Filters
Identifies emails by suspicious sender addresses, excessive keywords, known spam patterns, poor-reputation IPs. Uses Bayesian filters, deny/allow lists, and ML algorithms.
Malware Filtering
Scans email attachments and links for malware, preventing harmful content from reaching inboxes.
Encryption
Encrypts email to protect content from unauthorized access during transmission; can digitally sign emails to verify sender identity.
Authentication
Verifies sender identities to prevent email spoofing and impersonation.
Email Authentication Standards — SPF, DKIM, DMARC

SPF

Sender Policy Framework works at the level of the receiving email server. When an email is sent, the recipient's server checks the SPF record of the sender's domain to verify whether the sending server's IP address is authorized to send emails on behalf of that domain.

If not listed in the SPF record, the server may flag the email as suspicious, reject it, or take other actions. This helps prevent email spoofing and sender address forgery.

DKIM

DomainKeys Identified Mail works alongside SPF. DKIM uses cryptographic signatures to verify that an email was actually sent by the claimed domain and that the content was not altered in transit.

SPF alone has limitations — it does not travel with the email header when forwarded. DKIM provides an additional layer that remains with the message.

DMARC

Domain-based Message Authentication, Reporting, and Conformance works with both SPF and DKIM to provide comprehensive email security. DMARC tells receiving servers what to do when SPF or DKIM checks fail.

SPF is an effective tool in the fight against email spoofing and fraud, but it has limitations and should be used in conjunction with DKIM and DMARC for a more robust strategy.

Content Filter vs. Spam Filter

Spam Filters
Aim to reduce spam reaching a user's inbox. Focus on characteristics associated with spam: suspicious senders, excessive keywords, known spam patterns, poor-reputation IPs. Highly automated using algorithms and heuristics.
Content Filters
Target a broader range of content — not only spam but also inappropriate or sensitive data. Enforce content-related policies. Used in web filtering, email filtering, and DLP.

Modern SEG Evolution

Secure email gateways continue to evolve as threats evolve. Today, greater automation and machine learning are built into SEGs, alleviating demands on security operations centers.

In some cases, a SEG is integrated with other network security devices such as edge and segmentation firewalls and sandboxes.
